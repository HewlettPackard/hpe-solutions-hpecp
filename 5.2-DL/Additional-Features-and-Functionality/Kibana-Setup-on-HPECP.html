    # Kibana Setup on HPECP

    # Introduction

    Kibana is a free and open frontend application that sits on top of the Elastic Stack. It provides visualization capabilities where users can create bar, line and scatter plots, or pie charts and maps on top of large volumes of data

    Pre-requisites

    • SSH access to HPECP controller node

    # Flow Diagram

    Figure 73. Flow Diagram for Kibana setup on HPECP

    # Kibana Setup

    Kibana should be running by default inside epic-monitoring container under controller node. If Kibana is not running perform the following steps

    1. SSH to the controller node

    2. Search for epic-monitoring container and ssh to the container

    3. Change the permission of /etc/kibana/kibana.yml to 644

    4. Start the kibana service and verify the service is running fine

    5. Obtain the Kibana admin credentials

      Exit from the epic-monitoring container and run below commands in controller node. Output of the command provides the credentials to login to Kibana

    6. Access Kibana WEBGUI using controller_ip:5610

      Provide the credentials obtained from the step 5

    7. User can view the Kibana page as below

    8. Create index patterns

      Go to Management section, click on Index patterns and Click Create Index pattern

    9. Type any pattern or choose Wildcard "*" and in next step choose timestamp

    10. Once index pattern is created, click on Discover and choose the required index pattern

    11. Select Visualize and use various option to generate graph or visualize the data.

    12. Once visualization is created, save the chart

      Now all the visualization graph can be added to the dashboard

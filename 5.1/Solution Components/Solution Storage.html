# Solution Storage

NOTE

HPE Ezmeral Data Fabric for HPE Ezmeral Container platform 5.1 is in technical preview. To obtain software and additional details, contact hcpcustomerengg@hpe.com.

In this solution, we have leveraged HPE Ezmeral Data Fabric for Kubernetes. The components of HPE Ezmeral Data Fabric for Kubernetes are described at a high-level in Figure 7.

The HPE Ezmeral Data Fabric for Kubernetes provides the ability to run HPE Ezmeral Data Fabric services on top of Kubernetes as a set of pods. It allows you to create Data Platform Clusters for storing data and Tenants for running application such as spark jobs inside pods. Installing both the Tenant components and the Data Platform Cluster components, the HPE Ezmeral Data Fabric Platform runs as a fully native Kubernetes application. Deploying the Data Platform Cluster as part of the HPE Ezmeral Data Fabric for Kubernetes provides the following benefits:

  • Independent and elastic scaling of storage and compute

  • Ability to run different versions of Spark applications using the same data platform

  • Deployment of multiple environments with resource isolation and sharing as required

Figure 8. High-level HPE Ezmeral Data Fabric for Kubernetes architecture

By default, HPE Data Platform Clusters on Kubernetes require a minimum of 5 nodes with at least 16 Cores, 60 GB Memory, 500GB of storage available to the OS and 500GB of ephemeral storage. Additional 3X 500 GB disks for configuring storage for HPE Ezmeral Data Fabric for Kubernetes.
